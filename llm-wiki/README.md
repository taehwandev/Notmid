# notmid LLM Wiki

This folder is the short, task-oriented memory surface for LLM agents working on notmid.

Keep this folder for Notmid product facts, module inventory, route URLs, backend
contracts, and Firebase/open-source policy. Reusable Android architecture,
verification, ViewModel, data-flow, module-boundary, and agent workflow rules
belong in Tao Agent OS, not in `llm-wiki`.

Read only what is relevant:

- [notmid-overview.md](notmid-overview.md): product, audience, screens, naming.
- [module-map.md](module-map.md): current repo/module ownership inventory.
- [design-system.md](design-system.md): Notmid theme, tokens, Material3 wrappers, component rules.
- [routing-deeplinks.md](routing-deeplinks.md): Notmid route targets and canonical deep-link behavior.
- [platform-backend.md](platform-backend.md): server-first web/API/contracts direction.
- [firebase-open-source.md](firebase-open-source.md): Firebase/auth direction and secret handling.

Do not put API keys, Firebase service accounts, MCP credentials, or local-only config in this folder.
